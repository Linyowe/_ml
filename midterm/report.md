# 期中專案報告：喵塞塔石碑 (Meow-setta Stone) - AI 貓語翻譯機

**課程名稱**：機器學習
**專案類別**：用 LLM/AI 解讀其他語言（貓） / 參考：羅塞塔石碑

---
使用Gemini 3.1pro輔助編寫

## 1. 摘要 (Abstract)

本專案「喵塞塔石碑 (Meow-setta Stone)」受到古埃及「羅塞塔石碑」的啟發，旨在建立一個能夠將貓咪行為與聲音解讀為人類意圖的機器學習模型。由於目前缺乏大規模且標註完整的公開貓語資料集，本研究首先開發了一個具統計學意義的合成資料生成器，產生包含多維度特徵（聲音類型、持續時間、音頻、時間、環境狀態）的貓咪行為資料集。接著，我們採用了隨機森林分類器 (Random Forest Classifier) 進行監督式學習。實驗結果顯示，該模型在測試集上達到了 92.67% 的高準確率。最終，我們提供了一個實用的預測介面，能將預測出的分類意圖，包裝為生動的人類語言輸出。

## 2. 研究動機與背景 (Motivation & Background)

在人類與寵物（特別是貓）的互動中，經常會因為「語言不通」而產生誤解。有時貓咪喵喵叫是為了討食，有時卻只是想引起注意，或者是因為感到生氣或壓力。
參考了課程中提到的「用 LLM 解讀其他語言（例如外星人、嬰兒、海豚、貓）」，我們希望能打造一塊數位時代的「羅塞塔石碑」：
- **已知文字**：環境上下文與聲音的物理特徵（如頻率、長度）。
- **未知文字**：貓咪的真實心聲。
透過機器學習演算法的模式識別能力，我們試圖在兩者之間建立起精準的翻譯橋樑。

## 3. 資料搜集與方法 (Methodology)

### 3.1 合成資料集生成 (Synthetic Dataset Generation)
我們撰寫了 `generate_cat_data.py` 來建立包含 1,500 筆資料的合成資料集 `cat_language_dataset.csv`。
資料集中包含了以下特徵 (Features)：
1. **`sound_type` (類別特徵)**：Meow (喵叫), Purr (呼嚕聲), Hiss (嘶嘶聲), Trill (短促高音) 等。
2. **`duration_sec` (數值特徵)**：聲音持續時間，以常態分佈隨機生成。
3. **`pitch_hz` (數值特徵)**：音調頻率，反映聲音的高低。
4. **`time_of_day` (數值特徵)**：發生時間（0~23 時），反映作息規律。
5. **`bowl_empty` (二元特徵)**：飼料碗是否為空 (1=空, 0=有飼料)。

目標標籤 (Target Label) 設定為四種意圖 (Intention)：
- `Hungry` (肚子餓)
- `Pet_Me` (想被撫摸/討摸)
- `Angry` (生氣/警戒)
- `Open_Door` (要求開門)

### 3.2 資料預處理與模型建立
在 `cat_translator_model.py` 中，我們進行了以下步驟：
- 使用 `LabelEncoder` 將文字型態的類別特徵 (`sound_type` 與 `intention`) 轉換為機器可讀的數值特徵。
- 採用 80/20 比例切分訓練集與測試集。
- 選擇 **Random Forest (隨機森林)** 作為核心演算法。隨機森林是由多棵決策樹組成的集成學習 (Ensemble Learning) 方法，對於非線性的行為特徵能有極佳的分類效果，且對 Outlier 具有良好的抵抗力。

## 4. 實驗結果 (Experimental Results)

模型訓練完成後，我們在包含 300 筆資料的測試集上進行評估，結果如下：

| 類別 (Intention) | Precision (精確率) | Recall (召回率) | F1-Score |
| :--- | :--- | :--- | :--- |
| **Angry (生氣)** | 0.96 | 0.87 | 0.92 |
| **Hungry (肚子餓)** | 0.96 | 0.96 | 0.96 |
| **Open_Door (想出門)**| 0.92 | 0.90 | 0.91 |
| **Pet_Me (討摸摸)** | 0.89 | 0.97 | 0.93 |
| **Overall Accuracy** | - | - | **92.67%** |

### 結果分析
- 模型整體的準確率高達 93%，這證明了在有明確的上下文特徵（如 `bowl_empty` 與 `sound_type` 關聯性）時，機器學習能夠極度精準地判斷出背後的真實意圖。
- 尤其是 `Hungry` 的判斷 F1-Score 達到了 0.96，因為「碗是空的」且「發出喵聲」的特徵組合非常強烈且容易學習。

## 5. 系統實作展示 (Implementation Demo)

我們也在腳本中內建了 `translate_cat_behavior` 函式，可用來模擬實際應用場景：

**場景一：半夜三點、發出長長的喵聲、且飼料碗是空的**
- 預測意圖：`Hungry`
- 翻譯輸出：「人類！我的碗空了，立刻給我罐罐！」

**場景二：下午兩點靠近人、發出低頻呼嚕聲**
- 預測意圖：`Pet_Me`
- 翻譯輸出：「過來摸我，但只能摸三下，多一下我就咬你。」

## 6. 結論與未來展望 (Conclusion & Future Work)

本專案成功實作了一個「貓語羅塞塔石碑」，不僅展示了機器學習如何透過多維度特徵來解釋未知的行為模式，也提供了一個非常有趣且實用的應用情境。

**未來展望**：
1. **真實資料導入**：未來可透過部署麥克風與攝影機，蒐集真實貓咪的音頻與影像，並使用神經網路 (CNN) 萃取音頻特徵來取代目前的合成資料。
2. **導入 LLM 生成動態對話**：目前的翻譯輸出為固定字串，未來可結合大型語言模型 (LLM)，將分類出來的 `Intention` 當作 Prompt，讓翻譯機根據貓咪的「個性」生成獨一無二的生動台詞。
